# Pilot proposal — offering AI-guided STEM remediation to the TWAS community

**Proposed with / through the TWAS Young Affiliates Network (TYAN)**

> **Note on intent.** This is *not* a funding request. LMS Bridge is already built, free, and
> open-source. The aim is for TWAS/TYAN to **try it, then offer it to members** as a ready-made
> capacity-building tool — optionally hosted by TWAS and presented under a TWAS name. The "pilot"
> below is simply the lightweight first step that lets a few member-educators use it and produces
> evidence TWAS can point to when recommending it more widely.

## Summary

A short, low-cost pilot in which a few **TYAN-affiliated educators** use **LMS Bridge** — a free,
open-source, AI-guided adaptive-learning tool — to improve foundational concept mastery in an
introductory STEM course. The pilot uses **Sage**, LMS Bridge's standalone mode, so it needs **no
LMS and no IT department**: an instructor can start in minutes, and TWAS can host a shared,
TWAS-branded instance so members just sign in.

## Why this is a good fit for TWAS/TYAN

- **Mission fit:** strengthens science capacity in the developing world; supports SDG 4 (quality
  education); a reusable public good members can share South–South.
- **A member benefit, not a cost:** free and open-source (AGPL-3.0); no licensing fee, no
  procurement hurdle. TWAS can host one instance (or members self-serve via a browser).
- **Brandable:** the tool can run under a TWAS name/logo (e.g. "TWAS Learning") on a TWAS
  subdomain or a special link, while remaining the same upstream open-source project.
- **Generates evidence:** even a small pilot yields data suitable for a short report or a TWAS
  communication — useful when TWAS decides to recommend it to its wider membership.

## Research question

Does just-in-time, AI-guided remediation (retrieval practice + Socratic scaffolding) measurably
improve students' mastery of foundational STEM concepts compared with the same course taught
without it?

## Design (lightweight)

1. **Scope:** 1–2 instructors, one introductory course each (e.g. programming, digital logic,
   statistics), one term — or a focused 4–6 week module.
2. **Setup:** each instructor creates a Sage course, authors short quizzes whose questions are
   tagged to concepts, and shares a join code with students.
3. **Intervention:** when a student misses a concept, Sage automatically generates a guided
   tutoring session; the instructor sees who needs review.
4. **Optional comparison:** stagger the intervention by section or by week to compare outcomes.

## Measures

- Change in concept-level mastery (at-risk → developing → mastered).
- Quiz score improvement on re-attempts after remediation.
- Engagement: remediation sessions started/completed.
- Instructor and student feedback (short surveys).

## What it needs

- One or two willing instructors and their students.
- An AI provider key (or the deterministic mock for training/demo). The software is free; a modest
  amount of AI credit per student is the only variable cost. Options to keep this at/near zero:
  each institution brings its own key, use the non-AI features and mock mode, or apply for donated
  education/nonprofit credits from an AI provider.
- Light technical support from the project team. For a shared, TWAS-hosted instance, a small server
  is enough; for Sage used per-instructor, even that is optional.

## Timeline

- Weeks 0–2: confirm sites, brief instructors, set up courses.
- Weeks 3–N: run the course/module, collect data.
- Weeks N+1–N+3: analyze, write a short report, plan scale-up.

## Outputs

- A short results report and anonymized dataset.
- A reusable playbook other TYAN members can follow.
- Candidate material for a TWAS feature and a learning-science publication.

## Roles

- **Academic lead:** the TYAN affiliate at the host institution.
- **Participating instructors:** run the courses.
- **Project team (LMS Bridge):** onboarding, training, light support.

## From pilot to a TWAS member benefit

The pilot is near-zero-cost and asks for no funding. If it goes well, the natural next step is for
TWAS/TYAN to **offer the tool to the wider membership** — for example by hosting a single
TWAS-branded instance ("TWAS Learning") on a TWAS subdomain or special link, listing it among
member resources, and running an onboarding workshop. Because the project is open-source and free,
TWAS adopts it with no licensing cost and full control over its own instance and data.
